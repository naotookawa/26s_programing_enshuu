# 潜水艦ゲーム 仕様・利用ガイド

## 1. 概要

潜水艦ゲームは、人やAIが対戦できる5x5盤面のターン制CLIゲームです。

各プレイヤーは戦艦・巡洋艦・潜水艦を1隻ずつ配置します。手番では艦を移動するか、艦の周囲を攻撃します。攻撃で艦の耐久力を0にすると沈没し、相手の全艦を沈没させたプレイヤーが勝利します。

この文書では、下記の「絶対ルール」を実装仕様として扱います。古いルールと矛盾する場合は、この文書のルールを優先します。

## 2. 絶対ルール

1. 各プレイヤーはそれぞれ5x5のマス目上に戦艦、巡洋艦、潜水艦を配置する。位置は相手に伝えない。自分の艦同士を同じ場所に配置してはいけない。
2. 先行後攻を決める。
3. 手番が回ってきたプレイヤーは艦の移動か攻撃を行える。
4. 移動では、自分の艦1隻を縦横のどちらかにいくらでも動かすことができる。ただし、自分の他の艦がいる場所には動かせない。相手の艦と位置が重複してしまっても構わない。移動を行った場合はどの艦をどちらに何マス動かしたかを報告する。
5. 攻撃では、自分の艦がいる位置及び周囲1マスを攻撃することができる。この際どのマスを攻撃するかだけを報告すればよい。
6. 攻撃を受けたプレイヤーはマス目を調べる。自分の艦があれば耐久力を1減らし艦種と命中したことを報告する。耐久力が0になった艦は沈没し、今後ゲームには登場しない。
7. 耐久力は戦艦3、巡洋艦2、潜水艦1である。
8. 攻撃対象が自分の艦の周囲1マスであった場合はその艦の名前を報告する。
9. 以上をいずれかのプレイヤーの艦が全て沈没するまで繰り返す。最後まで艦が残っていたプレイヤーが勝利する。

## 3. 実行環境

- OS: ターミナルが使える環境
- 言語: Python 3.x
- 実行形式: CLIアプリケーション
- 通信: 不要
- 外部サーバー: 不要
- 外部Pythonパッケージ: 不要

確認済み環境:

- Python 3.13.0
- uv 0.7.2

## 5. ゲーム仕様

### 5.1 盤面

- 盤面サイズは5x5です。
- 列は `A`〜`E`、行は `1`〜`5` です。
- 座標は `A1`, `C3`, `E5` の形式で入力します。
- 内部表現では `row`, `col` を0〜4の整数で扱います。

### 5.2 艦

各プレイヤーは以下の3隻を持ちます。

| 記号 | 艦種 | 耐久力 |
| --- | --- | --- |
| `B` | 戦艦 | 3 |
| `C` | 巡洋艦 | 2 |
| `S` | 潜水艦 | 1 |

各艦は1マスのみを占有します。耐久力が0になった艦は沈没し、移動・攻撃・周囲報告・残機判定の対象から外れます。

### 5.3 初期配置

デフォルトでは、ゲーム開始時に各プレイヤーが自分の艦を手入力で配置します。

```text
Player A: 戦艦の位置を入力してください: A1
Player A: 巡洋艦の位置を入力してください: C3
Player A: 潜水艦の位置を入力してください: E5
```

同じ座標に複数の自艦を配置することはできません。

`--random-setup` を指定すると、各プレイヤーの艦をランダム配置します。`--auto-play` を指定した場合も初期配置はランダムになります。

### 5.4 先攻後攻

先攻はゲーム開始時に決めます。

- `--first-player A`: Player A が先攻
- `--first-player B`: Player B が先攻
- `--first-player random`: ランダムに決定

指定しない場合は `random` です。`--seed` を指定すると、ランダムな先攻決定も再現できます。

### 5.5 ターン進行

各ターンは次の順序で進みます。

1. 現在のプレイヤーの情報を表示する。
2. `attack` または `move` を選ぶ。
3. 選んだ行動を実行する。
4. 相手の全艦が沈没したか確認する。
5. 勝者がいなければ相手に交代する。

### 5.6 移動

移動では、自分の未沈没艦を1隻選び、東西南北のいずれかに1〜4マス移動します。5x5盤面なので、実装上の最大距離は4マスです。

移動の制約:

- 盤面外へ出る移動はできません。
- 移動後の座標に自分の未沈没艦がいる場合は移動できません。
- 自分の沈没艦の座標は、今後ゲームに登場しないため移動先として扱えます。
- 相手の艦との重なりは許可します。
- 移動経路上の衝突判定は行いません。

移動後、相手には以下だけが公開されます。

- 移動した艦種
- 移動方向
- 移動距離

移動前後の正確な座標は相手には公開されません。

### 5.7 攻撃

攻撃では、自分の未沈没艦を1隻選び、その艦がいる位置および周囲1マスを攻撃できます。攻撃可能範囲は、選んだ艦を中心とする最大3x3の範囲です。

攻撃時、攻撃者は攻撃座標を報告します。

防御側は攻撃座標を調べ、未沈没艦がいればその艦の耐久力を1減らします。

- 耐久力が残った場合: 艦種と命中、残り耐久力を報告します。
- 耐久力が0になった場合: 艦種と命中、沈没を報告します。
- 艦がなかった場合: 外れを報告します。

攻撃後、攻撃座標が防御側の未沈没艦の周囲1マスであった場合、その艦名を周囲報告として報告します。複数の未沈没艦が該当する場合はすべて報告します。攻撃座標そのものにいる艦は命中報告で扱うため、周囲報告には重複して含めません。攻撃で沈没した艦も、その攻撃後の周囲報告には含めません。

### 5.8 ログ・ヒント・公開情報

毎ターン、現在のプレイヤー視点で次の情報を表示します。

- 自分の盤面
- 自分の艦の位置・耐久力・沈没状態
- 相手の残機数
- これまで得られたヒント
- 直近の相手行動
- 自分の過去行動ログ

表示される履歴は、次のように役割を分けています。

- これまで得られたヒント: 自分の攻撃結果・周囲報告と、相手の移動公開情報
- 直近の相手行動: 相手が公開した攻撃座標・攻撃結果・周囲報告と、相手の移動公開情報
- 自分の過去行動ログ: 自分が実行した攻撃・移動

ヒント例:

```text
Turn 1: 攻撃 E1 -> 命中: 戦艦 耐久 2/3 / 周囲報告: なし
Turn 4: 攻撃 D1 -> 命中: 潜水艦 撃沈 / 周囲報告: 戦艦, 巡洋艦
Turn 6: 相手移動 -> 巡洋艦 north 2
```

相手行動例:

```text
Turn 3: 相手が E1 を攻撃。結果: 命中: 戦艦 耐久 1/3。周囲報告: なし。
Turn 5: 相手は 戦艦 を east に 2 マス移動しました。
```

### 5.9 勝利判定

相手の未沈没艦が0隻になった時点で、現在のプレイヤーが勝利します。

勝利時には、勝利バナー、敗者の全艦沈没メッセージ、最終盤面、行動ログ、ヒント履歴を表示します。

```text
============================================================
                    V I C T O R Y
------------------------------------------------------------
                         Player A の勝利！
                    全敵艦撃沈。作戦成功です。
                      勝者に盛大な拍手を！
============================================================
```

### 5.10 自動対戦

`--auto-play` を指定すると、Player A と Player B がどちらもCPUとして行動します。

自動対戦のCPUは、ゲームの動作確認用の簡易CPUです。対人戦の推理AIではなく、内部状態を使って次の順序で合法行動を選びます。

1. 自艦の攻撃範囲内に未沈没の敵艦がいる場合、その敵艦を攻撃する。
2. 攻撃できる敵艦がいない場合、未沈没の敵艦に最も近づく移動を選ぶ。
3. 移動できない場合、攻撃可能範囲からランダムに攻撃する。

`--seed` を指定すると、ランダム配置・先攻決定・自動行動を再現できます。`--max-turns` を指定すると、長引いた場合に指定ターンで終了します。

## 6. 遊び方

### 6.1 通常起動

```bash
uv run python -m submarine
```

### 6.2 ランダム配置

```bash
uv run python -m submarine --random-setup
```

### 6.3 先攻指定

```bash
uv run python -m submarine --first-player A
uv run python -m submarine --first-player B
uv run python -m submarine --first-player random
```

### 6.4 自動対戦

```bash
uv run python -m submarine --auto-play
```

再現性のある自動対戦:

```bash
uv run python -m submarine --auto-play --seed 1 --max-turns 120
```

| オプション | 意味 |

```text
=== Turn 1: Player A のターン ===

自分の盤面:
   A B C D E

2  . . . . .
3  . . C . .
4  . . . . .
```

命中して耐久力が残る場合:

```text

```

命中して沈没する場合:

```text

```

```text
外れました。
周囲報告: 攻撃座標の周囲1マスに敵艦はいません。
```

```text

```

相手には次のように公開されます。

```text
Turn 6: 相手移動 -> 巡洋艦 north 2
```

## 10. エラー処理

不正入力があってもゲームは終了せず、再入力を求めます。

主なエラー:

- 座標形式が不正
- 盤面外の座標
- 存在しない艦種
- 沈没済み艦の選択
- 攻撃可能範囲外への攻撃
- 不正な方向
- 不正な距離
- 盤面外への移動
- 自分の未沈没艦と重なる移動
- `--max-turns` が1未満

## 11. 検証方法

非対話の簡易テストを実行できます。

```bash
uv run python -B -m submarine.smoke_test
```

確認済み項目:

- 座標パース
- 座標表示
- 攻撃可能範囲取得
- 攻撃可能範囲判定
- 移動後座標計算
- 艦種と耐久力
- 命中時に耐久力を1減らすこと
- 耐久力0で沈没すること
- 沈没艦を周囲報告から除外すること
- 攻撃座標・結果・周囲報告を相手の公開イベントへ記録すること
- 自艦との重なり判定
- 自動対戦でCPUが入力なしに合法行動を選ぶこと
- `--auto-play --seed 1 --max-turns 120` で勝利判定まで実行できること

## 12. 現在の制約

- 画面クリアは行いません。
- 同じ端末を交代で使用します。
- ネットワーク対戦には対応していません。
- 矢印キーによる疑似GUI操作には対応していません。
- セーブ・ロード機能はありません。
- 人間対CPUモードはまだありません。
