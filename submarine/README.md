# 潜水艦ゲーム

人やAIが対戦できる、5x5盤面の潜水艦ゲームです。各プレイヤーは戦艦・巡洋艦・潜水艦を1隻ずつ配置し、艦の移動と攻撃を繰り返します。相手の艦をすべて沈没させたプレイヤーが勝利します。

## 実行方法

`submarine/` パッケージが見えるディレクトリで実行します。

```bash
uv run python -m submarine
```

ランダム配置で開始する場合:

```bash
uv run python -m submarine --random-setup
```

CPU同士で自動対戦させる場合:

```bash
uv run python -m submarine --auto-play
```

再現性のある自動対戦にしたい場合:

```bash
uv run python -m submarine --auto-play --seed 1 --max-turns 120
```

## 絶対ルール

- 各プレイヤーは5x5の盤面に、戦艦・巡洋艦・潜水艦を1隻ずつ配置します。
- 自分の艦同士を同じ場所に配置してはいけません。
- 相手の艦と同じ場所へ移動してもかまいません。
- 先攻後攻を決めてからゲームを開始します。指定しない場合はランダムです。
- 手番では、艦の移動か攻撃を1回行います。
- 移動では、自分の未沈没艦1隻を縦横いずれかに1マス以上動かします。盤面外や自分の未沈没艦がいる場所には移動できません。
- 移動した場合は、艦種・方向・距離を報告します。移動前後の正確な座標は相手には公開しません。
- 攻撃では、自分の未沈没艦1隻を選び、その艦の位置または周囲1マスを攻撃します。
- 攻撃した場合は、攻撃座標を報告します。
- 攻撃を受けた側は、攻撃座標に自分の未沈没艦があれば耐久力を1減らし、艦種と命中を報告します。
- 耐久力は戦艦3、巡洋艦2、潜水艦1です。
- 耐久力が0になった艦は沈没し、以後ゲームには登場しません。
- 攻撃座標が防御側の未沈没艦の周囲1マスであった場合、防御側はその艦名を周囲報告として報告します。攻撃座標そのものにいる艦は命中報告で扱うため、周囲報告には重複して含めません。
- いずれかのプレイヤーの艦がすべて沈没するまで続け、最後まで艦が残っていたプレイヤーが勝利します。

## 入力

- 行動: `attack`, `a`, `攻撃`, `move`, `m`, `移動`
- 艦: `B`, `C`, `S`, `戦艦`, `巡洋艦`, `潜水艦`
- 方向: `north`, `south`, `east`, `west`, `北`, `南`, `東`, `西`
- 座標: `A1`〜`E5`

## 起動オプション

| オプション | 意味 |
| --- | --- |
| `--random-setup` | 初期配置をランダムにする |
| `--auto-play` | CPU同士で自動対戦する |
| `--seed N` | ランダム配置・自動行動・先攻ランダム決定の乱数シードを指定する |
| `--max-turns N` | 自動対戦の最大ターン数を指定する |
| `--first-player A/B/random` | 先攻プレイヤーを指定する |

## 検証

非対話の簡易テストを実行できます。

```bash
uv run python -B -m submarine.smoke_test
```
